# ImageMonsters
**アップロード画像から“自分だけのモンスター”を生成し、育成・進化させるゲーム。**

## 概要
ユーザーが画像をアップロードすると、画像の特徴（色・形・雰囲気など）を解析して**固有のモンスター**を生成します。  
生成されたモンスターは、育成（経験値・トレーニング）や進化（フォーム変化）、バトル（PvE中心、将来的にPvPも）を通じて成長します。

> ※本作のゲーム性は「育成・調教・大会」など、いわゆる育成ゲーム（モンスターファーム系）のループを参考にしています。

---

## 主な機能
- 🖼️ **画像→モンスター生成**：アップロード画像を基に、見た目・属性・能力・設定を生成
- 🧬 **特性＆ステータス**：属性（火/水/風など）、レアリティ、スキル、性格を画像特徴から決定
- 🌱 **育成＆進化**：レベルアップ・進化条件達成でフォーム解放（幼生→成体→完全体）
- ⚔️ **バトル**：相性・スキルを軸にしたコマンドバトル（PvE）
- 📚 **図鑑（コーデックス）**：生成したモンスターの履歴、進化ルート、設定を保存
- 🔒 **プライバシー配慮**：画像保存方針（削除/保持/埋め込みのみ保存）を明確化

> 注意：本プロジェクトは開発中です。APIやデータ形式は変更される可能性があります。

---

## 仕組み（ざっくり）
1. **アップロード**
   - ユーザーが画像（jpg/png/webp）を投稿
2. **画像解析（Image → Text / Keywords）**
   - 画像の特徴をテキスト化し、重要度順にキーワードを最大10個抽出
3. **モンスター設計図（Blueprint）生成**
   - キーワードから、属性 / ボディタイプ / レアリティ / 基礎ステ / スキル / 寿命 を決定
4. **見た目生成（Text & Image → Character）**
   - キーワード + 元画像（参照）からキャラクターを生成
   - 進化表現（幼生→成体→完全体）のため、進化段階ごとにキーワードを絞り込み
5. **ゲーム性（育成ループ）**
   - 生成したモンスターを育成し、大会やミッションで強化素材・称号・図鑑を進める

## ゲーム性（設計メモ）
### 基本ループ
- **生成**：画像→モンスター入手（1体もの）
- **育成**：トレーニング（ステ増）/ 休養（コンディション回復）/ アイテム（補助）
- **大会・ミッション**：報酬（素材/通貨/称号）を得る
- **進化**：条件達成（Lv・特性・大会ランク・寿命など）でフォーム変化
- **収集**：図鑑（モンスター生成履歴）を埋めていく

### 育成パラメータ例
- `HP / ATK / DEF / SPD`
- `Condition`（疲労/やる気）
- `Lifespan`（寿命：育成期間の上限）
- `Traits`（性格・気質：トレーニング成功率、スキル発動率などに影響）

### 進化（幼生→成体→完全体）
- **幼生**：キーワードを広めに（雰囲気優先、弱いが伸びしろ）
- **成体**：主軸キーワードを固定（属性・体型が固まる）
- **完全体**：上位キーワードに絞る + 成長履歴を反映（育成方針で分岐も可能）
